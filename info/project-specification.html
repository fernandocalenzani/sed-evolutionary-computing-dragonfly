Resumo do Projeto

Componentes Principais

  1. Backend: Lida com a lógica de negócios e interage com o frontend, consumer e producer.
  2. Frontend: Interface do usuário que interage com o backend.
  3. Consumer/Producer: Responsável pela comunicação entre o hardware e o backend.
    • Consumer: Recebe dados do RabbitMQ e realiza pré-processamento.
    • Producer: Envia dados para o RabbitMQ.
    • Broker: Gerencia a comunicação assíncrona entre o Consumer e o Producer.
    • __main__.py: Ponto de entrada para o Consumer/Producer.
  4. IA (Inteligência Artificial): Módulo dedicado ao processamento inteligente.
    • Data: Armazena dados para treinamento e inferência.
    • Preprocessing: Lida com a preparação de dados.
    • Products: Modelos de IA específicos para diferentes finalidades (d_face, d_obj, r_action, r_emotion, r_face, t_objects).
  5. Libs: Bibliotecas e módulos de suporte.
    • Helpers: Funções auxiliares reutilizáveis.
    • Project: Módulo principal com funcionalidades compartilhadas.

Infraestrutura e Implantação

A infraestrutura pode ser implementada utilizando tecnologias como RabbitMQ para a comunicação assíncrona. Utilização de Docker para empacotar e distribuir as diferentes partes do projeto. Integração com serviços em nuvem, como Google Cloud Platform (GCP), para escalabilidade e facilidade de gerenciamento.

Configurações

  • As configurações do projeto podem ser gerenciadas através de arquivos de configuração.
  • Recomenda-se utilizar variáveis de ambiente para sensibilidade e configurações específicas do ambiente.

Escalabilidade

  • A arquitetura permite escalabilidade horizontal conforme a demanda aumenta.
  • Componentes individuais podem ser escalados independentemente, proporcionando flexibilidade.
